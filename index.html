Social media blogging app (repuddle)

This project was my introduction to web development. It is a fully developed social media site, with custom Vanilla JS and PHP solutions.

Screenshots

Features

  • User Profiles

    • Create and manage user accounts

    Engage with posts by liking, commenting, bookmarking and sharing

    Set and edit display name, username, bio and social links

    Profile types

    There are 3 different types - Verified, Writer and a generic profile

    Verified and Writer accounts have a dedicated badge that shows in front of the user's name, and have the ability to write articles, create and manage lounges.

  • Content Creation

    • Writers and Verified users can create two types of content - articles and short form posts

    Article page

    Articles are:

    Editable, shareable and search engine optimized.

    Categorized with a main category and tags.

    Lounge page

    Inspired by Naver's cafe section

    Lounges are pages which foster multiple types of so called boards, on which appear different types of posts, decided by the editors.

    For example, a community lounge can have boards like a Notice Board, Discussions, Announcements, etc. The "Notice Board" can be set up to contain only short-form posts, while the rest can contain articles with topics relevant only to the boards they are assigned to. Certain boards can only have content posted by editors or moderators of the lounge.

    The user selects which lounge and which board to post his article in the article creation page.

  • Search

    • The search functionality is as follows:

    Search results update on each input, with requests being throttled to 1000ms/r to preserve resources

    The "Trending" section on the home page is tied to the search logic, by saving the search bar input in the form of search terms in the database, once the user clicks on a result. In case of repetition, a column named "times_searched" is being incremented by 1.

  • SMTP Automated Emailing

    • I implemented an automated email confirmation system using SMTP (Simple Mail Transfer Protocol), and used the PHPMailer library for managing the email sending process. This library simplifies SMTP configuration and email sending in PHP

    Automated Emails are set up for account confirmation, user security and recovery.

  • Reporting

    • Users can report all user created content in a few clicks. Reports can be anonymous as well.

About

Ensuring the extension consistently removed various ad elements from YouTube pages was challenging. This included identifying and removing elements representing ads and handling YouTube's frequent updates which often changed the structure of the webpage. I also had to develop a solution to skip video ads by manipulating the video player's playback rate and duration.

One key challenge was ensuring the consistent skipping of video player ads. Initially I would detect whenever the 'Skip ad' button was available in the DOM and simulate a click on it, but that was inconsistent and required additional fallback measures like speeding up, muting and setting the opacity of the player to '0' whenever it didn't find that skip button. I remembered the fast forward extension i made was capable of also fast forwarding while ads were running, so I decided to implement that same logic and simulate a keyboard event to the 'ArrowRight' key with a value of "999999..", which would skip any ad that popped up.

This project enhanced my skills in DOM manipulation, event handling, and using MutationObserver for monitoring changes to the webpage.
